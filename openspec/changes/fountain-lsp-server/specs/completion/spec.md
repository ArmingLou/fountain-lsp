## 新增需求

### 需求:文本补全功能
LSP Server 必须提供文本补全功能，当用户输入特定触发字符时，自动提供候选补全项列表。

#### 场景:角色名补全触发
- **当** 用户在 Fountain 文档中输入字符 `@`
- **那么** LSP Server 必须触发补全请求，并返回角色名候选列表

#### 场景:补全项显示
- **当** 用户触发补全后收到候选列表
- **那么** 每个补全项必须包含：标签文本（角色名）、插入文本、详细描述（说话次数、出现场景数）

#### 场景:替换模式补全
- **当** 用户选择一个补全项且当前光标前存在 `@` + 前缀文本
- **那么** LSP Server 必须用选中的角色名完整替换 `@` 和前缀文本

#### 场景:续接模式补全
- **当** 用户选择一个补全项且光标位于 `@` 字符处
- **那么** LSP Server 必须在 `@` 之后直接插入角色名，不替换任何字符

#### 场景:补全提交方式
- **当** 用户通过按键（Enter/Tab）确认补全项
- **那么** LSP Server 必须将选中的文本插入到文档中，并关闭补全列表

#### 场景:补全取消
- **当** 用户按下 Escape 键或点击补全列表外部区域
- **那么** LSP Server 必须取消当前补全会话，关闭补全列表

#### 场景:空补全结果
- **当** 用户触发补全但文档中不存在任何角色名
- **那么** LSP Server 必须返回空补全列表，而不是错误

### 需求:触发字符配置
LSP Server 必须声明支持 `@` 字符作为补全触发字符。

#### 场景:触发字符注册
- **当** 客户端发送 `initialize` 请求
- **那么** LSP Server 必须在 `completionProvider` 的 `triggerCharacters` 字段中包含 `@`
