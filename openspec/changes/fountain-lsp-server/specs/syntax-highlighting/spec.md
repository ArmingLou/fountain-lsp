## 新增需求

### 需求:Tree-sitter 语法高亮
LSP Server 必须提供基于 Tree-sitter 的语法高亮功能，通过 highlights.scm 定义规则。

#### 场景:场景标题高亮
- **当** 编辑器请求语法高亮且文档包含场景标题（INT./EXT. 开头）
- **那么** 场景标题文本必须使用 `@title` 高亮类别显示

#### 场景:角色名高亮
- **当** 编辑器请求语法高亮且文档包含角色名
- **那么** 角色名文本必须使用 `@type` 高亮类别显示

#### 场景:对话高亮
- **当** 编辑器请求语法高亮且文档包含对话内容
- **那么** 对话文本必须使用 `@string` 高亮类别显示

#### 场景:括号内动作指示高亮
- **当** 编辑器请求语法高亮且文档包含括号内动作指示
- **那么** 括号内文本必须使用 `@emphasis` 高亮类别显示

#### 场景:动作描述高亮
- **当** 编辑器请求语法高亮且文档包含动作描述
- **那么** 动作描述文本必须使��� `@comment` 高亮类别显示

#### 场景:转场高亮
- **当** 编辑器请求语法高亮且文档包含转场文本（CUT TO: 等）
- **那么** 转场文本必须使用 `@keyword` 高亮类别显示

#### 场景:标题页高亮
- **当** 编辑器请求语法高亮且文档包含标题页
- **那么** 标题页中的键值对必须使用 `@property` 高亮类别显示

#### 场景:章节标题高亮
- **当** 编辑器请求语法高亮且文档包含章节标题（# ## ###）
- **那么** 章节标题必须使用 `@heading` 高亮类别显示

### 需求:LSP Semantic Tokens 增强
LSP Server 必须支持通过 Semantic Tokens 提供增强的语法高亮。

#### 场景:语义令牌类型声明
- **当** 客户端发送 `initialize` 请求
- **那么** LSP Server 必须在 `semanticTokensProvider` 中声明支持的 token 类型（sceneHeading、character、dialogue、action、parenthetical、transition）

#### 场景:语义令牌范围
- **当** 客户端发送 `textDocument/semanticTokens/full` 请求
- **那么** LSP Server 必须返回文档中所有匹配 token 类型的字符范围及其类型分类

#### 场景:语义令牌增量更新
- **当** 文档发生变更且客户端请求语义令牌增量更新
- **那么** LSP Server 必须只返回变更部分的 token 增量数据

#### 场景:语义令牌修饰符
- **当** 返回 Semantic Token
- **那么** 必须根据 token 类型正确设置修饰符（如 dialogue 的 `readonly` 修饰符）

### 需求:高亮性能
语法高亮必须在合理的时间内完成。

#### 场景:首次加载
- **当** 打开一个 10000 行的 Fountain 文档
- **那么** 语法高亮必须在 2 秒内完成

#### 场景:增量高亮
- **当** 用户编辑文档（单行修改）
- **那么** 增量高亮必须在 100ms 内完成
