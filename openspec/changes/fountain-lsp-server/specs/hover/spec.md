## 新增需求

### 需求:悬停提示功能
LSP Server 必须提供悬停提示功能，当用户将鼠标悬停在文档中的特定元素上时显示上下文相关信息。

#### 场景:场景标题悬停
- **当** 用户将鼠标悬停在场景标题（Scene Heading）上
- **那么** LSP Server 必须返回包含以下信息的 Markdown 内容：
  - 场景时长（格式："时长: Xm Ys"）
  - 位置信息（INT./EXT.）
  - 时间（日/夜：morning/afternoon/evening/night/dusk/dawn）
  - 出现场景号列表（如 "🎬 #1, 🎬 #2, 🎬 #3"）
  - 如果是分割场景引用（#${var}#），显示引用数量（如 "(🎬 #1) was split into 3"）

#### 场景:角色名悬停
- **当** 用户将鼠标悬停在角色名上
- **那么** LSP Server 必须返回包含以下信息的 Markdown 内容：
  - 角色名（格式："👤 **角色名**"）
  - 角色描述（如果有，来自 characterDescribe）
  - 出现场景号列表（如 "🎬 #1, 🎬 #2, 🎬 #3"）
  - 出现场景总数（如 "appears in (5 scenes):"）

#### 场景:地点悬停（场景位置）
- **当** 用户将鼠标悬停在场景标题中的地点名称上
- **那么** LSP Server 必须返回包含以下信息的 Markdown 内容：
  - 地点名（格式："🏠 **地点名**"）
  - 出现场景号列表
  - 出现场景总数
  - 如果是分割场景引用，显示引用数量

#### 场景:对话内容悬停
- **当** 用户将鼠标悬停在对话文本上
- **那么** LSP Server 必须返回包含以下信息的 Markdown 内容：
  - 对话时长（格式："时长: Xs"）
  - 所属角色

#### 场景:动作描述悬停
- **当** 用户将鼠标悬停在动作描述（Action）上
- **那么** LSP Server 必须返回包含以下信息的 Markdown 内容：
  - 动作类型（"动作描述"）
  - 该行在剧本中的位置（"第 X 行"）

#### 场景:转场文本悬停
- **当** 用户将鼠标悬停在转场文本上
- **那么** LSP Server 必须返回包含以下信息的 Markdown 内容：
  - 转场类型（如 "CUT TO:", "FADE IN:", "DISSOLVE TO:"）

#### 场景:无悬停内容
- **当** 用户将鼠标悬停在不包含有效 Fountain 元素的文本上
- **那么** LSP Server 必须返回空悬停结果，而不是错误

### 需求:悬停响应格式
悬停响应必须遵循 LSP 规范。

#### 场景:Markdown 格式
- **当** LSP Server 返回悬停内容
- **那么** 内容必须使用 Markdown 格式（`MarkupContent` 的 `kind` 为 `markdown`）

#### 场景:悬停范围
- **当** 返回悬停内容
- **那么** 必须包含正确的 `range` 字段，指示悬停适用的字符范围

#### 场景:多行悬停
- **当** 悬停内容需要显示多行信息
- **那么** 必须使用 Markdown 换行符（`\n`）分隔各行

### 需求:悬停性能
悬停提示必须在合理的时间内显示。

#### 场景:响应时间
- **当** 用户触发悬停
- **那么** LSP Server 必须在 200ms 内返回悬停内容
